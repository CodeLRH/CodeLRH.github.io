（所有题目与前几版完全不重复，答案详细、基础、不偏，适合面试背诵）

一、通信原理（3题）

1. 单工、半双工、全双工通信的区别

答案：

  • 单工通信：消息只能单方向传输，一方固定发送，一方固定接收，例如广播、电视、遥控。
  • 半双工通信：通信双方都可以收发消息，但不能同时收发，例如对讲机。
  • 全双工通信：通信双方可以同时收发消息，信道双向同时畅通，例如手机、固定电话。

2. 什么是多径效应？会造成什么影响？

答案：
信号经过多条不同路径到达接收端，每条路径的时延、衰减随时间不断变化，这种现象称为多径效应。

  • 主要影响：使信号产生快衰落，造成码间干扰，使波形失真，严重时会导致误码。

3. 什么是白噪声、高斯白噪声？

答案：

  • 白噪声：功率谱密度在整个频域内均匀分布的理想化噪声，类似白光频谱。
  • 高斯白噪声：幅度服从高斯分布，且功率谱均匀的噪声，是通信系统中最主要、最常用的噪声模型，如热噪声。

二、模拟电子（3题）

4. 什么是放大电路的静态工作点（Q点）？作用是什么？

答案：
放大电路在无输入信号时，晶体管的基极电流、集电极电流、管压降的数值称为静态工作点（Q点）。

  • 作用：为晶体管提供合适的直流偏置，保证信号放大时工作在放大区，避免出现饱和失真和截止失真。

5. 什么是饱和失真、截止失真？

答案：

  • 饱和失真：静态工作点过高，晶体管进入饱和区，输出波形底部被削平。
  • 截止失真：静态工作点过低，晶体管进入截止区，输出波形顶部被削平。

注：两者都属于非线性失真，会使输出波形无法还原输入信号。

6. 什么是负反馈？对放大电路有哪些作用？

答案：
将输出信号的一部分或全部，以相反极性送回输入端，削弱净输入信号，称为负反馈。
主要作用：

  1. 稳定放大倍数；
  2. 减小非线性失真；
  3. 展宽通频带；
  4. 改变输入电阻和输出电阻；
  5. 提高电路稳定性。

三、数字电子（3题）

7. 什么是译码器？什么是编码器？

答案：

  • 编码器：将有效输入信号转换为对应的二进制代码输出，例如优先编码器。
  • 译码器：将输入的二进制代码翻译为对应状态的输出信号，例如3-8线译码器、七段显示译码器。

8. 什么是计数器？有什么用途？

答案：
能够记忆输入时钟脉冲个数的时序逻辑电路称为计数器。

  • 用途：计数、定时、分频、产生节拍脉冲、数字运算、控制时序等。

9. 什么是移位寄存器？主要功能是什么？

答案：
不仅能存储二进制代码，还能使代码在时钟控制下逐位移位的寄存器。

  • 功能：串并数据转换、数据延时、脉冲移位、序列信号产生。

四、高频电子线路（4题｜重点加强）

10. 小信号谐振放大器的主要特点是什么？

答案：
以LC并联谐振回路作为负载，具有选频+放大双重作用；属于窄带放大器；工作在甲类线性状态，失真小；主要用于高频小信号的选频放大。

11. 振荡器与放大器的根本区别是什么？

答案：

  • 放大器：必须有输入信号才能工作，输出与输入相关。
  • 振荡器：不需要外加输入信号，依靠电路正反馈自动将直流能量转换为特定频率的交流信号输出。

12. 调频中的预加重和去加重的目的是什么？

答案：

  • 最终目的：提升高频段输出信噪比，改善音质。
  • 预加重：发送端提升高频信号幅度。
  • 去加重：接收端解调后衰减高频信号，同时把高频噪声一起衰减（最终使信号不变、噪声降低，提高输出质量）。

13. 振幅调制、解调、混频的共同点是什么？

答案：

  1. 三者都属于频谱线性搬移电路；
  2. 时域上都表现为信号相乘；
  3. 频域上实现频谱不失真搬移；
  4. 都可以用相乘器 + 滤波器实现。

五、电子基础（2题｜基础不偏）

14. 什么是串联谐振和并联谐振？

答案：

  • 串联谐振：RLC串联时总阻抗最小、电流最大，呈纯电阻性。
  • 并联谐振：RLC并联时总阻抗最大、电压最大，呈纯电阻性。

注：谐振时电路无功功率为零，总阻抗为纯电阻。

15. 什么是线性失真和非线性失真？

答案：

  • 线性失真：不产生新的频率分量，仅幅度/相位发生变化，由幅频、相频特性不理想引起。
  • 非线性失真：产生新的频率分量，由器件非线性引起，如饱和失真、截止失真。

通信类考研复试核心15题（通信原理/模电/数电/高频）

2026-03-24
学习笔记

科目分布：通信原理3 + 模电3 + 数电3 + 高频电子4 + 单片机1 + C语言1（无信号）

一、通信原理（3题）

1. 数字通信系统相比模拟通信系统的优点

答案：

  1. 抗干扰能力强，噪声不累积；
  2. 便于进行各种数字信号处理；
  3. 易于集成化，设备体积小、功耗低；
  4. 便于实现多路复用，提高频带利用率；
  5. 可靠性高，可通过信道编码进行检错与纠错；
  6. 易于实现综合业务数字网（ISDN）。

2. 信源编码与信道编码的作用

答案：

  • 信源编码：减少信号冗余度，进行数据压缩，提高通信系统的有效性。
  • 信道编码：加入监督码元，实现检错和纠错，提高通信系统的可靠性。

3. 什么是码间串扰？如何消除？

答案：
码间串扰是指前后码元的波形发生展宽、拖尾，对当前码元的抽样判决造成干扰，导致误码。
消除方法：使基带系统的冲激响应仅在本码元抽样时刻取最大值，在其他码元抽样时刻均为0，满足奈奎斯特第一准则，即可实现无码间串扰传输。

二、模拟电子技术（3题）

4. PN结的单向导电性原理

答案：

  • PN结正向偏置（P正、N负）时，内电场被削弱，多子扩散运动加强，形成较大正向电流，PN结导通；
  • PN结反向偏置（P负、N正）时，内电场被加强，多子扩散被抑制，只有极小的反向漏电流，PN结截止。
    因此电流只能单方向通过，称为单向导电性。

5. 晶体管实现放大的外部条件

答案：
必须满足：发射结正向偏置，集电结反向偏置。
只有满足该偏置条件，晶体管才能工作在放大区，实现电流放大作用。

6. 差分放大电路的主要功能

答案：
差分放大电路的核心特点是：放大差模信号，抑制共模信号。
最大作用是有效抑制零点漂移（温度漂移），广泛用作集成运放的输入级。

三、数字电子技术（3题）

7. 组合逻辑电路与时序逻辑电路的区别

答案：

  • 组合逻辑电路：无记忆功能，输出仅由当前输入决定，不含触发器。
  • 时序逻辑电路：有记忆功能，输出由当前输入 + 电路原来状态共同决定，包含触发器。

8. 什么是竞争与冒险？如何消除？

答案：
由于信号经过不同路径到达门电路的时间不同，导致输出瞬间出现干扰毛刺，称为竞争冒险。
消除方法：

  1. 修改逻辑设计，增加冗余项；
  2. 引入选通脉冲；
  3. 输出端并联小滤波电容。

9. 触发器的基本功能

答案：
触发器是时序逻辑电路的基本单元，具有0/1两种稳定状态，能够存储一位二进制信息，具备记忆功能。

四、高频电子线路（4题 · 重点加强）

10. 通信系统为什么要采用调制？

答案：

  1. 使低频基带信号搬移到高频频段，便于天线辐射，实现远距离传输；
  2. 减小天线尺寸，便于设备小型化；
  3. 实现频分复用（FDM），多个信号共用信道，提高频带利用率；
  4. 提高系统抗干扰能力与传输性能。

11. LC并联谐振回路的特性与作用

答案：

  • 特性：谐振时回路阻抗最大，且为纯电阻，相移为0。
  • 作用：选频、滤波、阻抗匹配、波形放大，是高频放大器与振荡器的核心部件。

12. 高频谐振功率放大器为何工作在丙类？

答案：
丙类工作状态的导通角小于180°，晶体管仅在信号部分周期导通，管耗小、效率高，适合大功率高频功率放大。
依靠LC谐振回路的滤波作用，仍可输出不失真的正弦波形。

13. 超外差接收机中混频器的作用

答案：
将接收到的不同载频高频信号变换为固定频率的中频信号。

  • 优点：中频固定且较低，容易实现高增益、高选择性放大，显著提高接收机的灵敏度、稳定性和抗干扰能力。

五、单片机原理（1题）

14. 单片机中断响应的条件

答案：

  1. 总中断允许位 EA = 1；
  2. 对应中断源的允许位为1；
  3. 无更高级别中断正在服务；
  4. 当前指令已经执行完毕。

六、C语言程序设计（1题）

15. 全局变量与局部变量的区别

答案：

  • 全局变量：定义在函数外部，存储在静态存储区，作用域贯穿整个程序，生命周期与程序相同。
  • 局部变量：定义在函数内部，存储在栈区，仅在函数内部有效，函数调用结束后释放。

注意：两者可以重名，当重名时，局部变量会屏蔽全局变量。

电池管理系统(BMS)进阶：动力电池 SOC 估算核心算法深度拆解

2026-03-12
专业探索

在新能源汽车与大规模储能系统中，电池管理系统（BMS）是保障安全与效率的大脑。而在 BMS 庞杂的软件架构中，**电池荷电状态（State of Charge, SOC）**的精确估算，始终是学术界与工业界共同死磕的“硬骨头”。

SOC 估算之所以困难，是因为电池本质上是一个高度非线性、时变且包含复杂电化学反应的“黑盒”。我们无法放置一个传感器在电池内部去直接“称量”剩余的锂离子，只能通过外部可测的端口特性（电压、电流、温度）来逆推内部状态。这个过程伴随着极化效应、滞回特性、以及随生命周期（SOH）不断漂移的内阻和容量。

本文将从基础原理出发，深度拆解目前主流的 SOC 估算算法，并探讨其在底层嵌入式系统中的工程落地挑战。

一、 传统标定方法：理想与现实的差距

1. 安时积分法 (Coulomb Counting)

安时积分法是目前工业界应用最广泛的兜底算法，其数学表达极为直观：

SOC(t)=SOC(t0)−1Cn∫t0tη⋅i(τ)dτSOC(t) = SOC(t_0) - \frac{1}{C_n} \int_{t_0}^{t} \eta \cdot i(\tau) d\tau SOC(t)=SOC(t0​)−Cn​1​∫t0​t​η⋅i(τ)dτ

其中，SOC(t0)SOC(t_0)SOC(t0​) 是初始荷电状态，CnC_nCn​ 是当前电池的最大可用容量，η\etaη 是库仑效率，i(τ)i(\tau)i(τ) 是充放电电流。

局限性剖析：
尽管原理简单，但安时积分法是一个典型的“开环”系统。它面临三大致命缺陷：

  • 积分漂移：电流传感器的零点漂移和测量白噪声在积分环节会被无限累加，时间越长误差越大。
  • 初值依赖：如果系统断电重启时无法获取准确的 SOC(t0)SOC(t_0)SOC(t0​)，整个估算轨迹将永远平移偏离真实值。
  • 容量衰减盲区：分母上的 CnC_nCn​ 会随着电池老化而动态变小，若不结合 SOH 估算进行实时修正，算出的 SOC 将失去物理意义。

2. 开路电压法 (OCV)

由于电池的开路电压（OCV）与 SOC 存在着相对稳定的单调非线性映射关系，OCV 查表法常被用来修正安时积分的初值。
局限性剖析：

  • 电池必须静置极长时间（通常数小时）才能消除极化电压，测得真实的 OCV。
  • 磷酸铁锂（LFP）电池在 20%∼80%20\% \sim 80\%20%∼80% 区间存在极长的 OCV 平坦期，几十毫伏的电压测量误差就可能导致百分之十几的 SOC 误判。

二、 基于模型的闭环观测器：控制理论的降维打击

为了解决开环系统的累积误差，现代 BMS 普遍引入了基于等效电路模型（ECM）和现代控制理论的闭环估计架构。

1. 建立状态空间模型

我们通常采用二阶戴维南（Thevenin）或双极化（DP）模型来模拟电池的欧姆内阻、电化学极化和浓差极化。将其离散化后，可以建立如下标准的非线性状态空间方程：

状态方程（描述系统内部演进）：

观测方程（描述内部状态到外部测量的映射）：

yk=h(xk,uk)+vky_k = h(x_k, u_k) + v_k yk​=h(xk​,uk​)+vk​

其中，状态变量 xk=[SOCk,Up1,k,Up2,k]Tx_k = [SOC_k, U_{p1,k}, U_{p2,k}]^Txk​=[SOCk​,Up1,k​,Up2,k​]T，输入 uku_kuk​ 是电流，输出 yky_kyk​ 是端电压，wkw_kwk​ 和 vkv_kvk​ 分别代表过程噪声和测量噪声矩阵。

2. 扩展卡尔曼滤波 (EKF) 与矩阵理论的交响

[Image of Extended Kalman Filter flow chart]

由于电池的 OCV-SOC 曲线 h(xk,uk)h(x_k, u_k)h(xk​,uk​) 具有强非线性，经典的线性卡尔曼滤波无法直接使用。**扩展卡尔曼滤波（EKF）**通过在一阶泰勒展开点求解雅可比矩阵（Jacobian Matrix），实现了系统的局部线性化。

在 EKF 的迭代过程中，协方差矩阵的推演是核心。算法通过不断预测先验状态估计误差协方差矩阵 Pk−P_{k}^-Pk−​，并结合实时的电压预测误差（新息）来计算卡尔曼增益 KkK_kKk​：

Kk=Pk−HkT(HkPk−HkT+R)−1K_k = P_{k}^- H_k^T (H_k P_{k}^- H_k^T + R)^{-1} Kk​=Pk−​HkT​(Hk​Pk−​HkT​+R)−1

最终完成状态的后验更新。EKF 的绝妙之处在于它构建了一个误差修正闭环：即使初始 SOC 给定错误，算法也能在数十个迭代周期内，凭借矩阵的收敛特性，将 SOC 估算值强行“拉拽”回真实轨迹。

注：为了应对更高阶的非线性，无迹卡尔曼滤波（UKF）通过 Unscented 变换构造 Sigma 点阵列来传递协方差，避免了复杂的雅可比矩阵求导，在强非线性区间表现更佳。

三、 数据驱动与前沿探索：打破物理模型的桎梏

无论是二阶还是高阶 RC 模型，都无法完美刻画电池全生命周期的非线性动态。近年来，基于纯数据驱动的机器学习算法成为了 SOC 估算领域的研究热点。

1. 时间序列的深度学习方案

将连续采样的电压 VVV、电流 III、温度 TTT 作为一个时间窗口内的多维特征向量，输入到循环神经网络（RNN）及其变体（如 LSTM 或 GRU）中。
LSTM 通过遗忘门和记忆门机制，天然契合电池“当前状态依赖于历史充放电路径”的特性。只要训练数据包含了足够丰富的温度梯度和老化状态，神经网络可以拟合出一个极其强悍的高维非线性映射，直接输出 SOC。

2. 多算法融合（Hybrid Algorithms）

目前的工程前沿不再拘泥于单一路径，而是走向融合。例如：

  • 神经网络 + EKF：利用轻量级的神经网络在线实时预测 EKF 中的过程噪声协方差矩阵 QQQ 和测量噪声协方差矩阵 RRR，实现自适应卡尔曼滤波（AEKF）。
  • 基于数据驱动的参数辨识：使用递推最小二乘法（RLS）或粒子群算法（PSO），在线实时更新等效电路模型中的 R0,Rp,CpR_0, R_p, C_pR0​,Rp​,Cp​ 参数，对抗电池老化带来的模型失配。

四、 算法的终极考验：嵌入式系统的工程落地

在 Simulink 或 Python 环境下跑通算法仅仅是第一步。真正将这些复杂的矩阵运算和网络模型塞进 BMS 的微控制器（MCU）中，才是巨大的工程挑战。

  1. 算力与存储瓶颈：MCU（如基于 ARM Cortex-M4/M7 内核的芯片）的 RAM 和 Flash 极其有限。直接部署浮点矩阵求逆或庞大的 LSTM 权重网络，极易引发堆栈溢出。
  2. C 代码生成与定点化：为了提高运行效率，往往需要将模型生成的 C 代码进行定点化（Fixed-point）处理，或者利用特定的 DSP 指令集进行加速。如何在降低计算精度的同时保证滤波器不发散，是算法工程师的核心竞争力。
  3. 实时性要求：BMS 的主循环周期通常在 10ms-100ms 级别，复杂的矩阵运算必须在单个周期内执行完毕，这对算法的轻量化提出了极为严苛的要求。

结语

从基础的积分累计，到复杂的信号处理与矩阵理论，再到前沿的机器学习，SOC 估算技术依然在快速迭代。在底层逻辑与工程实现的不断碰撞中，我们正在离那颗完美的“透明电池”越来越近。

家常炖猪蹄教程：软糯香甜，一看就会！

2026-03-11
生活日常

周末去超市买到了新鲜的猪蹄，顺手记录一下这道家常炖猪蹄的做法。步骤简单，厨房新手也能轻松搞定！

🛒 准备食材

  • 主料：猪蹄
  • 配菜：根据个人喜好准备（如黄豆、莲藕、土豆等）
  • 香料：葱、姜、花椒、八角、香叶
  • 调料：料酒、十三香、盐、白砂糖、鸡精（或味精）、食用油
  • 灵魂用水：温水

🍳 制作步骤

第一步：焯水去腥

  1. 猪蹄冷水下锅。
  2. 放入切好的葱段、姜片，倒入适量料酒去腥。
  3. 水烧开后，继续煮 3 到 4 分钟，煮出表面的白沫。
  4. 将白沫撇掉，捞出猪蹄沥干水分备用。

第二步：爆香炒制

  1. 起锅烧油，油热后放入葱、姜、花椒，小火炒出香味。
  2. 倒入焯好水的猪蹄，翻炒至表面微微发黄。
  3. 期间加入适量十三香和盐。（小贴士：这时候也可以微微放一点白砂糖提鲜）。

第三步：加水慢炖

  1. 炒香后，向锅中加入温水（注意一定要加温水，冷水会让肉质紧缩变柴），水量没过猪蹄。
  2. 放入香叶和八角，大火烧开。
  3. 水开后，放入准备好的配菜，转中小火盖上锅盖慢炖，直到猪蹄炖熟软烂。

第四步：调味出锅

  1. 在炖煮的后半段，根据口味加入白砂糖。建议先少放一点，后面不够再加。
  2. 炖熟后尝一下汤汁，如果盐味不够此时再补一点盐。
  3. 快出锅的时候，撒入一点鸡精或者味精提鲜。
  4. 大火稍微收一下汁，就可以出锅啦！

💡 总结小贴士：
炒猪蹄时一定要炒到微黄，加水时一定要用温水，调味时本着“少量多次”的原则尝着加，这样炖出来的猪蹄绝对软糯入味！

智能空地协同消防系统——2023年电赛实战总结

1. 项目概况

在 2023 年全国大学生电子设计竞赛中，针对复杂环境下救援人员难以抵达的痛点，我们设计并实现了一套智能空地协同消防系统。该系统由一台智能自主无人机和一台地面消防车组成，通过视觉识别与无线通讯实现高效联动。

所获荣誉：2023年全国大学生电子设计大赛 山东省赛区本科组二等奖。

2. 核心硬件架构

本系统利用了多种嵌入式开发工具和硬件模块进行协同：

  • 主控芯片：采用 STM32 系列单片机，负责底层逻辑控制与运动补偿。
  • 视觉大脑：搭载 OpenMV 摄像头，利用其集成的图像处理算法进行火情实时检测。
  • 硬件设计：利用 Altium Designer 完成了系统核心 PCB 板的绘制与电路打样。
  • 调试工具：开发过程中重度使用 Keil、VS Code 以及串口调试助手进行逻辑优化。

3. 工作流程与逻辑实现

3.1 发现火情（无人机端）

无人机在高空进行巡检，通过 OpenMV 捕捉特定的颜色阈值（模拟火源）。一旦锁定目标，系统会计算出火源相对于无人机的方位信息。

3.2 协同通讯

无人机通过无线通讯模块将火情坐标实时传输至地面消防车。

3.3 快速抵达（消防车端）

地面消防车接收到指令后，结合自身的传感器数据进行路径规划，以最快速度避障并到达模拟现场执行救援任务。
